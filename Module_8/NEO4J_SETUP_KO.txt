## 옵션 A — Neo4j Aura (Free tier)

Aura는 Neo4j의 관리형 클라우드 데이터베이스입니다. Free tier로 이 과정을 충분히 수행할 수 있습니다.

### Step A1 — 계정 생성

1. [https://neo4j.com/cloud/aura/](https://neo4j.com/cloud/aura/) 열기
2. 가입 또는 로그인.
3. 새 **AuraDB Free** 인스턴스 생성(Graph Data Science 기능이 특별히 필요하지 않다면 AuraDS가 아님).

### Step A2 — 연결 정보 저장

인스턴스가 준비되면:

1. **Download credentials**를 클릭하거나 연결 정보를 복사.
2. 다음을 기록:
  - **Connection URI** (`neo4j+s://`로 시작)
  - **Username** (`neo4j`)
  - **Password** (한 번만 표시 — 지금 저장)

### Step A3 — Neo4j Browser 열기

1. Aura 콘솔에서 인스턴스의 **Open** / **Query** 클릭.
2. 스모크 테스트 실행:

```cypher
RETURN "Aura connection OK" AS message;
```

### Step A4 — LLM Knowledge Graph Builder(웹 앱)와 함께 사용

1. [https://llm-graph-builder.neo4jlabs.com/](https://llm-graph-builder.neo4jlabs.com/) 열기(또는 [Neo4j Labs — LLM Graph Builder](https://neo4j.com/labs/genai-ecosystem/llm-graph-builder/) 링크).
2. Aura URI, 사용자명, 비밀번호를 붙여넣기(또는 다운로드한 자격 증명 파일을 끌어다 놓기).
3. 샘플 문서를 업로드하고 **Generate Graph** 실행.

### Step A5 — Jupyter 노트북과 함께 사용

연결 셀 실행 전 환경 변수 설정(플레이스홀더 교체):

```bash
export NEO4J_URI="neo4j+s://YOUR_INSTANCE.databases.neo4j.io"
export NEO4J_USERNAME="neo4j"
export NEO4J_PASSWORD="your-password-here"
export NEO4J_DATABASE="neo4j"
```

Windows (PowerShell):

```powershell
$env:NEO4J_URI="neo4j+s://YOUR_INSTANCE.databases.neo4j.io"
$env:NEO4J_USERNAME="neo4j"
$env:NEO4J_PASSWORD="your-password-here"
$env:NEO4J_DATABASE="neo4j"
```

## 옵션 B — Neo4j Desktop (Windows, macOS, Linux)

Neo4j Desktop은 로컬 데이터베이스, 플러그인, Neo4j Browser를 관리하는 무료 애플리케이션입니다.

### Step B1 — 다운로드 및 설치

1. [https://neo4j.com/download/](https://neo4j.com/download/)에서 다운로드
2. 운영체제용 Neo4j Desktop 설치.
3. Desktop 실행 후 로그인(무료 계정).

### Step B2 — 로컬 데이터베이스 생성

1. **Create instance** (또는 **New** → **Instance**) 클릭.
2. 이름 지정, 예: `kg-llm-course`.
3. 기억할 **password** 설정.
4. Neo4j 버전 **5.15 이상** 선택.
5. 인스턴스 시작(상태가 **Running**이 되어야 함).

### Step B3 — APOC 활성화(아직 안 된 경우)

1. 인스턴스 열기 → **Plugins** → 가능하면 **APOC** 설치.
2. 또는 Desktop 문서에 따라 인스턴스 설정 / `apoc.conf`에 추가.

### Step B4 — 노트북용 연결 정보


| 설정  | 일반적인 값            |
| -------- | ------------------------ |
| URI      | `neo4j://localhost:7687` |
| Username | `neo4j`                  |
| Password | (인스턴스 비밀번호) |
| Database | `neo4j`                  |


**Open** → Neo4j Browser에서 테스트:

```cypher
RETURN "Desktop connection OK" AS message;
```

### Step B5 — Desktop과 LLM Graph Builder

호스팅 LLM Graph Builder는 **공개적으로 접근 가능한** 데이터베이스에 연결할 수 있습니다. 기본 Desktop 인스턴스는 보통 **localhost만** 허용하므로:

- **노트북의 프로그래밍 방식 추출**(LangChain + Neo4j)을 사용하거나,
- Docker로 [llm-graph-builder](https://github.com/neo4j-labs/llm-graph-builder)를 로컬 배포(아래 옵션 C + LLM Graph Builder 섹션 참고).

## 옵션 C — Docker로 Neo4j

Desktop 없이 깨끗하고 재현 가능한 Neo4j 서버가 필요할 때 Docker를 사용합니다.

### Step C1 — Docker 설치

- **Windows / macOS**: [Docker Desktop](https://www.docker.com/products/docker-desktop/)
- **Linux**: Docker Engine + Docker Compose plugin

확인:

```bash
docker --version
docker compose version
```

### Step C2 — Neo4j 5.x 실행 (Linux + Docker에서 검증됨)

모든 명령은 **KMOU_Course** 저장소 루트에서 실행하세요.

#### C2a — 데이터 폴더 준비

```bash
cd /path/to/KMOU_Course

mkdir -p Module_8/neo4j-course-data
sudo chown -R 7474:7474 Module_8/neo4j-course-data
```

Linux에서 Docker 내 Neo4j는 사용자 `7474`로 실행됩니다. `chown` 없이는 컨테이너가 종료 코드 **1** 또는 **70**으로 종료될 수 있습니다.

#### C2b — 컨테이너 시작 (과정 기본값)

**강한 비밀번호**를 사용하세요(`neo4j/neo4j` 사용 금지 — Neo4j 5.x는 약한 비밀번호로 시작하지 못하는 경우가 많음).

```bash
sudo docker rm -f neo4j-kg-course 2>/dev/null

sudo docker run -d \
  --name neo4j-kg-course \
  -p 7474:7474 -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/CourseNeo4j2026! \
  -e NEO4J_PLUGINS='["apoc"]' \
  -e NEO4J_server_memory_heap_initial__size=512m \
  -e NEO4J_server_memory_heap_max__size=1G \
  -v "$(pwd)/Module_8/neo4j-course-data:/data" \
  neo4j:5.26-community
```

시작 로그를 **`Started.`**가 보일 때까지 확인:

```bash
sudo docker logs -f neo4j-kg-course
```

**`Started.`**가 보이면 로그 추적 중단(첫 실행 시 APOC 다운로드로 **1–3분** 걸릴 수 있음).

컨테이너 실행 확인:

```bash
sudo docker ps --filter name=neo4j-kg-course
curl -I http://127.0.0.1:7474
```

| 포트 | 서비스                           |
| ---- | --------------------------------- |
| 7474 | Neo4j Browser (HTTP)              |
| 7687 | Bolt 프로토콜 (드라이버, 노트북) |

Browser 열기: [http://127.0.0.1:7474](http://127.0.0.1:7474) (또는 [http://localhost:7474](http://localhost:7474)).

**로그인**

- Connect URL: `neo4j://localhost:7687`
- Username: `neo4j`
- Password: `CourseNeo4j2026!` (`NEO4J_AUTH`와 일치해야 함)

> 다른 값을 사용한다면 `NEO4J_AUTH`와 `Module_8/.env`의 비밀번호를 함께 변경하세요 — 반드시 일치해야 합니다.

### Step C3 — 환경 변수

`Module_8/.env.example`을 `Module_8/.env`로 복사하고 설정:

```bash
export NEO4J_URI="neo4j://localhost:7687"
export NEO4J_USERNAME="neo4j"
export NEO4J_PASSWORD="CourseNeo4j2026!"
export NEO4J_DATABASE="neo4j"
```

### Step C4 — 중지 및 제거 (완료 후)

```bash
docker stop neo4j-kg-course
docker rm neo4j-kg-course
```

해당 폴더를 삭제하지 않는 한 데이터는 마운트된 볼륨에 유지됩니다.

## 문제 해결

### Docker (옵션 C): `http://localhost:7474`가 비어 있거나 로드되지 않음

로컬 머신에서 다음 확인을 **순서대로** 진행하세요:

#### 1) 컨테이너가 실행 중인가?

```bash
sudo docker ps -a --filter name=neo4j-kg-course
```

**STATUS**가 `Up X minutes`이고 포트 `0.0.0.0:7474->7474/tcp`여야 합니다.

상태가 `Exited` 또는 `Restarting`이면 로그 확인:

```bash
sudo docker logs neo4j-kg-course --tail 80
```

#### 2) 흔한 해결 — 볼륨 권한 (Linux)

로그에 `/data`에 대한 `Permission denied`가 있으면, bind mount 폴더가 root 소유인데 컨테이너 내부 Neo4j는 사용자 `7474`로 실행됩니다.

**KMOU_Course** 저장소 루트에서:

```bash
mkdir -p Module_8/neo4j-course-data
sudo chown -R 7474:7474 Module_8/neo4j-course-data
sudo docker restart neo4j-kg-course
```

1–2분 대기 후 [http://127.0.0.1:7474](http://127.0.0.1:7474) 열기.

#### 3) 포트가 이미 사용 중

`docker run`이 “port is already allocated”로 실패하면 다른 프로세스가 7474/7687을 사용 중:

```bash
sudo docker stop neo4j-kg-course
sudo docker rm neo4j-kg-course
# Or change host ports: -p 7475:7474 -p 7688:7687 and use http://localhost:7475
```

#### 4) 컨테이너 `Exited (1)` 또는 `Exited (70)`

| 종료 코드 | 흔한 원인 |
|-----------|----------------|
| **1** | 볼륨 쓰기 불가 (`chown -R 7474:7474`로 수정) |
| **70** | 약한 비밀번호(`neo4j/neo4j`) 또는 볼륨의 손상/부분 데이터 |

**컨테이너를 깨끗하게 재생성** (검증된 동작 명령):

```bash
cd /path/to/KMOU_Course

sudo docker rm -f neo4j-kg-course 2>/dev/null
sudo rm -rf Module_8/neo4j-course-data
mkdir -p Module_8/neo4j-course-data
sudo chown -R 7474:7474 Module_8/neo4j-course-data

sudo docker run -d \
  --name neo4j-kg-course \
  -p 7474:7474 -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/CourseNeo4j2026! \
  -e NEO4J_PLUGINS='["apoc"]' \
  -e NEO4J_server_memory_heap_initial__size=512m \
  -e NEO4J_server_memory_heap_max__size=1G \
  -v "$(pwd)/Module_8/neo4j-course-data:/data" \
  neo4j:5.26-community

sudo docker logs -f neo4j-kg-course
```

**`Started.`**가 보이면 로그 추적 중단 후 Browser 열기.

#### 5) Neo4j Browser 로그인

- Connect URL: `neo4j://localhost:7687` (또는 `bolt://localhost:7687`)
- Username: `neo4j`
- Password: `NEO4J_AUTH`와 동일(과정 기본값: `CourseNeo4j2026!`)

#### 6) 빠른 HTTP 테스트

```bash
curl -I http://127.0.0.1:7474
```

서비스가 올라오면 HTTP `200` 또는 `302`가 예상됩니다. `connection refused`이면 컨테이너가 아직 listen하지 않거나 실행 중이 아닙니다.

#### 7) APOC로 첫 부팅이 느려짐

로그에서 APOC 다운로드가 실패하면, 기본 이미지 확인을 위해 일시적으로 `NEO4J_PLUGINS` **없이** 시작(Step C2b와 동일한 비밀번호·메모리 설정 유지). Browser 로그인이 되면 `NEO4J_PLUGINS='["apoc"]'`를 다시 추가.

### `Unable to connect` / `ServiceUnavailable`

- 인스턴스가 **Running**인지 확인(Desktop / Aura / Docker).
- **URI** 스킴 확인: Aura는 `neo4j+s://`; 로컬 Docker/Desktop은 보통 `neo4j://`.
- 방화벽: 로컬 Bolt용 포트 **7687**이 열려 있어야 함.

### `Authentication failed`

- Aura 콘솔 또는 Desktop 인스턴스 설정에서 비밀번호 재설정.
- Username은 거의 항상 `neo4j`.

### `Unknown database`

- Aura Free는 보통 데이터베이스 이름 `neo4j`.
- 직접 생성하지 않았다면 사용자 정의 데이터베이스 이름을 설정하지 마세요.

### APOC 프로시저 누락

- APOC 플러그인 설치(Desktop Plugins 탭 또는 Docker `NEO4J_PLUGINS`).
- 설치 후 데이터베이스 재시작.

### LLM 추출 결과 그래프가 비어 있음

- 구조화 JSON에 모델이 너무 작음 — 더 큰 Ollama 모델 또는 OpenAI `gpt-4o-mini` 시도.
- 텍스트가 너무 짧음 — `Module_8/data/`의 샘플 파일 사용.
- JSON 파싱 오류는 노트북 셀 출력 확인.

### SSL 인증서 오류 (Aura)

- Aura의 정확한 URI 사용(`neo4j+s://...`).
- `neo4j` Python 드라이버 업데이트: `pip install -U neo4j`.
